# Tutorials

Hand-held lessons. You learn pREST by doing, end to end, in a safe setup we
control. Each tutorial finishes with you having made something real work.

Do them in order. Each one builds on the last.

| # | Tutorial | You will |
|---|---|---|
| 1 | [Turn a table into a REST API](getting-started.md) | serve pREST, seed a table, GET / POST / filter it |
| 2 | [Require a login](secure-an-endpoint.md) | gate a table behind JWT, mint a token, get a 200 |
| 3 | [Save and run a SQL script](save-a-query.md) | author a `.sql` file, call it as an endpoint |

These are **tutorials**, not reference. For "what does this key do?" see
`docs/reference.md`. For "why is it built this way?" see `docs/architecture.md`.
For "how do I do my own task?" see `CONTRIBUTING.md`.
